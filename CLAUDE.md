# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm test                          # run all 10 test suites (~1,704 tests)
node scoring_engine.test.js       # run a single suite (replace filename as needed)
npm run build                     # generate dist/ from data/hotels.json
node site_builder.js --verbose    # build with per-page output
```

Zero external npm dependencies — no `npm install` needed.

## Architecture

The site is a Node.js static site generator. One command (`node site_builder.js`) reads hotel data and writes a complete deployable site to `dist/`.

**Build pipeline:**
```
data/hotels.json
  → scoring_engine.js       (score each hotel per persona, 0–100)
  → explanation_engine.js   (generate written descriptions)
  → block_assembler.js      (assemble HTML blocks: hero, cards, CTAs, FAQs)
  → static_page_renderer.js (render blocks → full HTML pages with all CSS inline)
  → dist/                   (static files, deployed to Cloudflare Pages on push)
```

`hallucination_guard.js` and `confidence_enforcer.js` are called inside `explanation_engine.js` to prevent fabricated claims and enforce hedging.

**Two kinds of pages — do not confuse them:**

| Kind | Source | How it's built |
|---|---|---|
| Dynamic pages | `data/hotels.json` | Generated entirely by the pipeline (persona, hotel, region, comparison pages) |
| Static/informational pages | `pages/*.html` | Written by hand, copied to `dist/` verbatim by site_builder |

Adding a new static page requires two steps:
1. Create `pages/<slug>.html`
2. Add an entry to `STATIC_PAGE_SPECS` in `site_builder.js` (for the sitemap) and to `getRelatedGuides()` in `static_page_renderer.js` (so the guide appears in related sections across the site)

**Six traveller personas** drive persona page generation and hotel ranking weights:
`luxury` · `honeymoon` · `family` · `wellness` · `remote_work` · `value_luxury`

**Data source priority at build time:**
1. Airtable live sync (if `AIRTABLE_API_KEY` + `AIRTABLE_BASE_ID` env vars are set)
2. `data/hotels.json` — committed snapshot; used in local dev and CI without secrets
3. `integration_harness.js` — hardcoded fallback dataset

## Key Invariants

- **Never modify `data/hotels.json` without explicit user instruction.** It is the source of truth for all hotel scores and affiliate links.
- **Never fabricate hotel data, review scores, or affiliate links.** Each affiliate URL must be verified from the dataset: `h._affiliate_links[0].booking_url`.
- **`dist/` is never committed.** Cloudflare Pages builds it fresh from source on every push.
- **`_status: "inactive"` hotels are filtered from all page generation** — do not remove this filter in `site_builder.js`.
- Every affiliate CTA must have `affiliate_disclosure: true` in its block payload — the renderer throws if missing.
- All user-provided strings are HTML-escaped via `esc()` before output — never bypass this.
- Slugs are generated by `_slugify()` which strips accented characters (`"Géran"` → `"g-ran"`). This is intentional.
- Maximum 1 new informational page per SEO agent run.

## Content Rules

- White-hat SEO only. No thin content, no keyword stuffing.
- Ambient/decorative images in `assets/images/ambient/` are **not tied to any specific hotel**. Never associate them with a listing or location.
- Affiliate links use `rel="noopener sponsored"`. Every page with affiliate CTAs must include a disclosure.
- The hero slideshow in `index.html` runs `assets/js/ambient-visuals.js` — Ken Burns animation is JS-driven (not CSS-class-toggled) using `void el.offsetWidth` reflow flush to force restart on each cycle.

## CSS / Frontend

All CSS for generated pages lives in `static_page_renderer.js` inside `generateHead()`. The homepage (`index.html`) has its own inline `<style>` block plus `assets/css/global.css`. Changes to generated page design go in `static_page_renderer.js`; changes to the homepage go in `index.html` and `global.css`.

## Deployment

Push to `main` → Cloudflare Pages auto-deploys within ~90 seconds. Build command: `node site_builder.js`. Output directory: `dist`. No force-push, no `--no-verify`.
