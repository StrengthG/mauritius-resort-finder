This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm test                          # run all 16 test suites
npm run test:<suite>              # run one suite by name: scoring | hallucination | confidence | explanation | assembler | renderer | sync | builder | admin
node scoring_engine.test.js       # run any single *.test.js file directly
npm run build                     # generate dist/ from data/hotels.json
node site_builder.js --verbose    # build with per-page output
npm run sync                      # pull live data from Airtable → data/hotels.json
npm run admin                     # start the admin Express server (port 3001)
node seo_campaign_dashboard.js    # terminal backlink campaign dashboard
node seo_campaign_dashboard.js --html   # export HTML SEO report
node seo_outreach.js              # backlink outreach pipeline summary
node seo_outreach.js --priority   # show top 5 next outreach actions
node seo_outreach.js --type=guest_post --status=not_started  # filter by type/status
node seo_prospect_discovery.js    # discover new backlink prospects
node seo_prospect_scorer.js       # score and rank discovered prospects
node agents/run_pipeline.js       # run full Extract→Sort→Upload agent pipeline
node agents/run_pipeline.js --dry-run   # validate without pushing
```

Zero external npm dependencies for the static site generator — no `npm install` needed for builds. The admin panel (`admin/`) does require `npm install`.

## Architecture

The site is a Node.js static site generator. One command (`node site_builder.js`) reads hotel data and writes a complete deployable site to `dist/`.

**Build pipeline:**
```
data/hotels.json + data/hotel-images.json + data/hotel-coordinates.json
  → scoring_engine.js       (score each hotel per persona, 0–100)
  → phrase_library.js       (2,000+ editorial phrase templates, consumed by explanation_engine)
  → explanation_engine.js   (generate written descriptions; calls hallucination_guard + confidence_enforcer)
  → hotel_content_engine.js (long-form content blocks, consumed by explanation_engine)
  → block_assembler.js      (assemble HTML blocks: hero, cards, CTAs, FAQs)
  → static_page_renderer.js (render blocks → full HTML pages with all CSS inline)
  → hotel_image_engine.js   (image sitemap + CSS gradient placeholders when photos are absent)
  → dist/                   (static files, deployed to Cloudflare Pages on push)
```

`hallucination_guard.js` and `confidence_enforcer.js` are called inside `explanation_engine.js` to prevent fabricated claims and enforce hedging.

`data/hotel-images.json` stores per-hotel image metadata (hue data, alt text). `data/hotel-coordinates.json` stores lat/lng used for the map page. Neither is generated by the pipeline — edit them directly if image or coordinate data needs updating.

**Two kinds of pages — do not confuse them:**

| Kind | Source | How it's built |
|---|---|---|
| Dynamic pages | `data/hotels.json` | Generated entirely by the pipeline (persona, hotel, region, comparison pages) |
| Static/informational pages | `pages/*.html` | Written by hand, copied to `dist/` verbatim by site_builder |

Adding a new static page requires two steps:
1. Create `pages/<slug>.html` — use `node scripts/apply-template.js` to scaffold a page with the standard dark-luxury CSS and nav already wired up
2. Add an entry to `STATIC_PAGE_SPECS` in `site_builder.js` (for the sitemap) and to `getRelatedGuides()` in `static_page_renderer.js` (so the guide appears in related sections across the site)

**Six traveller personas** drive persona page generation and hotel ranking weights:
`luxury` · `honeymoon` · `family` · `wellness` · `remote_work` · `value_luxury`

**Data source priority at build time:**
1. Airtable live sync (if `AIRTABLE_API_KEY` + `AIRTABLE_BASE_ID` env vars are set)
2. `data/hotels.json` — committed snapshot; used in local dev and CI without secrets
3. `integration_harness.js` — hardcoded fallback dataset

## Admin Panel (`admin/`)

A separate Express/EJS/SQLite app for hotel CRUD, build triggers, and audit logs. Runs independently of the static site generator.

```bash
npm run setup-admin    # create the SQLite DB and tables (run once)
npm run seed-admin     # seed initial hotel records from data/hotels.json
npm run admin          # start server on ADMIN_PORT (default 3001)
```

Routes: `auth` (login/logout/CSRF), `hotels` (CRUD), `build` (trigger a site rebuild), `audit` (change log), `users` (user management). Sessions are stored in SQLite via `connect-sqlite3`.

The admin server can also be deployed to Railway — config is in `railway.toml` (start command: `node admin/server.js`).

## Cloudflare Pages Functions (`functions/api/`)

Serverless functions deployed alongside the static site:

- `chat.js` — **Big Dodo** AI chat assistant. Uses Cloudflare Workers AI (free tier, no external key). Requires the `AI` binding enabled in the Cloudflare dashboard (Pages → Settings → Functions → AI Bindings → Variable name: `AI`).
- `contact.js` — Contact form handler; uses `RESEND_API_KEY` (set as a Cloudflare dashboard secret, not in `.env`).
- `_hotel_data.js` — Static hotel data bundle shared by the above functions.

The functions use ES module syntax (`import`), unlike the rest of the codebase which uses CommonJS (`require`).

## Agent Pipeline (`agents/`)

Three-agent pipeline for adding hotels from raw research to production:

1. **ExtractAgent** (`extract_agent.js`) — reads raw research input, writes `data/Extract.md`
2. **SortAgent** (`sort_agent.js`) — structures extracted data, writes `data/Sort.md`
3. **UploadAgent** (`upload_agent.js`) — validates, rebuilds, runs tests, commits, and pushes

Pipeline state is persisted in `data/state.json`. Use `--force` to reprocess from scratch or `--dry-run` to build without pushing.

## Hotel Images

Images live under `dist/assets/images/hotels/{hotel_id}/` and are **not committed** — they must be present before build or the engine renders CSS gradient placeholders. Convention:

```
hero.webp        1200×800  (required for hotel detail pages)
gallery-1.webp   800×600
gallery-2.webp   800×600
gallery-3.webp   800×600
gallery-4.webp   800×600
thumb.webp       400×300   (used on ranking/persona card listings)
```

Drop WebP files into the correct path and rebuild — placeholders are replaced automatically with no code changes.

## Additional Engines

- **`search_engine_client.js`** — pure search algorithm (no DOM/FS); generates `dist/search-index.json` at build time and is mirrored in `assets/js/search.js` for the browser (no bundler).
- **`social_card_engine.js`** — generates SVG `og:image`/`twitter:image` cards per hotel. Cache at `data/social-card-cache.json`; cards only regenerate when hotel data changes.
- **`ga4_trending_engine.js`** — pulls GA4 engagement signals, caches daily to `data/trending-cache.json`, falls back to rating-derived defaults when the API is unavailable.
- **`hotel_content_engine.js`** — generates long-form hotel content blocks (used within `explanation_engine.js`).

## Key Invariants

- **Never modify `data/hotels.json` without explicit user instruction.** It is the source of truth for all hotel scores and affiliate links.
- **Never fabricate hotel data, review scores, or affiliate links.** Each affiliate URL must be verified from the dataset: `h._affiliate_links[0].booking_url`.
- **`dist/` is never committed.** Cloudflare Pages builds it fresh from source on every push.
- **`_status: "inactive"` hotels are filtered from all page generation** — do not remove this filter in `site_builder.js`.
- Every affiliate CTA must have `affiliate_disclosure: true` in its block payload — the renderer throws if missing.
- All user-provided strings are HTML-escaped via `esc()` before output — never bypass this.
- Slugs are generated by `_slugify()` which strips accented characters (`"Géran"` → `"g-ran"`). This is intentional.
- Maximum 1 new informational page per SEO agent run.

## Content Rules

- White-hat SEO only. No thin content, no keyword stuffing.
- Ambient/decorative images in `assets/images/ambient/` are **not tied to any specific hotel**. Never associate them with a listing or location.
- Affiliate links use `rel="noopener sponsored"`. Every page with affiliate CTAs must include a disclosure.
- The hero slideshow in `index.html` runs `assets/js/ambient-visuals.js` — Ken Burns animation is JS-driven (not CSS-class-toggled) using `void el.offsetWidth` reflow flush to force restart on each cycle.

## CSS / Frontend

All CSS for generated pages lives in `static_page_renderer.js` inside `generateHead()`. The homepage (`index.html`) has its own inline `<style>` block plus `assets/css/global.css`. Changes to generated page design go in `static_page_renderer.js`; changes to the homepage go in `index.html` and `global.css`.

## Deployment

Push to `main` → Cloudflare Pages auto-deploys within ~90 seconds. Build command: `node site_builder.js`. Output directory: `dist`. No force-push, no `--no-verify`.

The Dodo SEO Agent prompt lives at `docs/agents/dodo-seo-agent.md` — paste its contents into a new Claude conversation to run an SEO build session.
